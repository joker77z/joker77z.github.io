매일마다 좋은 개발 글 읽기

🧑🏼‍💻 읽으면 좋을 대상 : 이동 중 시간을 효율적으로 사용하고 싶은 주니어 개발자

오늘 소개하는 공부 방법은 내가 개발을 막 제대로 배우기 시작했을 당시부터 현재까지 꽤 오랜시간 이어온 버릇이다. 개발 공부를 시작했을 무렵 공부하는 시간이 부족했다. 인천에서부터 강남, 성수를 왔다갔다하다보니 그 시간에 개발이 도움될만한 것으로 무엇이 있을까 찾다보니, 좋은 글을 찾으면 북마크를 해놓는 버릇이 생겼다. 나는 북마크 해놓은 좋은 글들을 요즘 TIL 소재로 활용 중이다. TIL은 쓰고 싶은데 어떤 걸 써야할지 모르겠다면 이 방법을 추천한다.

그렇다면 좋은 글은 어떻게 찾을까?

좋은 글 찾기

카카오 오픈톡방

카카오 오픈톡방에는 여러 개발자방이 있다. 이 중에서 추천하고 싶은 방은 잡담이 많지 않은 방이다. 내가 알고 있는 오픈 카톡방은 출퇴근길 개발읽기방이다. 프론트, 백엔드, 데브옵스 영역 구분없이 글들이 올라오지만 양질의 글이 올라올 때가 많아서 아주 좋다. 요즘은 프론트 외에도 백엔드, 데브옵스 영역에도 궁금한 것들이 한개 씩 생기고 있는데, 그 때 지나가면서 봤었던 다른 영역의 좋은 글을 보면 도움이 될 때가 있었다. 프론트 오픈방에서 얻은 정보보다 여기서 얻은 정보들이 더 많다. 지금은 이 오픈카톡방을 제외한채 다른 곳들은 다 나갔다.

크롬 메인

난 핸드폰에서 크롬을 사용하는데 개발 관련하여 많이 검색하다보니 추천 글로 개발 관련 글들이 메인에 올라온다. 나의 호기심을 자극할만한 그런 글들을 추천해줘서 유용하다.

Velog 메인

Velog 메인에서도 정말 다양한 정보를 얻을 수 있다. 나같은 경우에는 Velog에서 앞으로 쓸 포스트 영감을 많이 얻는 편이다. 이유는 그럴만한게 사람들이 좋아요나 댓글이 많이 달린 것들이 메인에 올라오기 때문에 나 역시도 그런 글들을 보며 많은 생각을 하게 된다.

스터디 디스코드

함께 프론트엔드를 공부했었던 분들이 있는 디스코드에 좋은 정보를 공유해주어 바로바로 북마크를 해놓는 편이다. 여러분도 스터디를 진행하고 정보를 함께 공유할 수 있는 마음맞는 사람들을 만난다면 좋은 정보들을 얻을 수 있을 것이다. 혼자 좋은 지식들을 찾기는 어렵기 때문에 큰 도움이 되는 것 같다.

자 이제 좋은 글을 찾아놓고 북마크를 해놨다면 매일 저녁이나 아침 일찍 TIL을 써보자 :)

TIL 쓰기

TIL을 쓰기 위해서는 깃 레포에서 바로 작성하는 방법이 있고, 나처럼 markdown 앱에서 글을 써서 push하는 방법이 있다. 나는 typora를 사용하고 있는데 markdown형식으로 쓰면 notion처럼 바로 미리보기 할 수 있는 것이 장점이다. 이 때 단점은 매번 글을 쓰기 위해서 터미널을 키고 TIL 디렉토리로 이동하고, 글을 작성하고 나면 add-commit-push해야 한다는 것이었다. 이런 비용들을 줄이기 위해 터미널에서 alias를 이용하여 til만 쳤을 때 바로 TIL디렉토리로 이동하게 했고 add-commit-push를 git p만 치면 한방에 push되게 해서 지금은 매우 효율적인 시스템을 만들었다.

git p만 해도 add-commit-push하기

매번 커밋 내용을 적을 필요가 없기 때문에 이모티콘을 넣어줬다. 만약, 매번 커밋 내용을 넣고 싶다면 이모티콘대신 $@를 넣으면 된다! 그러면 git p -m "커밋 내용" 이런 형식을 사용할 수 있다.

git config --global alias.p '!f() { git add -A && git commit -m "✨" && git push; }; f'

til만 쳐도 디렉토리 이동하기

mac

mac에서는 iterm을 사용하고 있는 사용자가 많은 것이기 때문에 iterm을 기준으로 설명한다.

> 1. alias를 설정할 수 있는 파일로 진입한다.
vi ~/.zshrc

> 2. i키를 눌러서 편집상태를 on하고 alias를 찾아 아래와 같이 .git파일이 있는 디렉토리로 설정한다.
alias til="cd Documents/til/TodayILearned"

> 3. :wq로 저장하고 빠져나온뒤 아래 명령어로 터미널에 바로 반영해준다.
source ~/.zshrc

window

window유저라면 C:\Program Files\Git\etc\profile.d/aliases.sh 파일에서 alias설정을 하면 git bash툴에서는 정상적으로 동작한다.

하지만, hyper같은 것을 사용했다면 alias가 동작안할 수 있다. 이럴 때는 아래 두 가지를 설정해주면 정상적으로 될 것이다. 나도 window를 사용할 때 이문제로 고생했다.

config: {
  ...
  shellArgs: ['--command=usr/bin/bash.exe', '-l', '-i'],
  ...
}
config: {
  ...
  env: {TERM: 'cygwin'},
  ...
}
