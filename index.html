구글의 Core Web Vital이란?

구글의 코어 웹 바이탈은 웹 페이지 로딩 속도, 모바일 친화성, 세이프 브라우징, 암호화(HTTPS적용 여부), 방해요소 여부 등과 같은 웹 콘텐츠 이용자의 사용자 경험을 측정하는 많은 요인 중에서 구글이 중요하다 강조하는 3가지 메트릭스 LCP(Largest Contentful Pain), FID(First Input Delay), CLS(Cumulative Layout Shift)를 중요시 하는 개념입니다.

  • LCP(Largest Contentful Pain)는 가장 큰 콘텐츠가 렌더링되는데 걸리는 시간입니다. 일반적으로 이미지 또는 동영상이거나 큰 블록 수준의 텍스트 요소입니다.
  • FID(First Input Delay)는 사용자가 페이지와 처음 상호작용 했을 때 브라우저가 반응하기까지 걸리는 시간입니다.
  • CLS(Cumulative Layout Shift)는 페이지의 레이아웃이 움직인 거리의 정도를 측정합니다. 페이지의 레이아웃이 예상하지 못한 경우에 자주 발생하게 되면 사용자 경험은 좋지 않게 됩니다.

기준

좋음 개선 필요 느림
LCP 2.5초 이하 4초 이하 4초 초과
FID 100밀리초 이하 300밀리초 이하 300밀리초 초과
CLS 0.1 이하 0.25 이하 0.25 초과

LCP의 큰 콘텐츠를 로드하는데 걸리는 시간, FID의 반응 시간, CLS의 레이아웃 변화의 정도. 다 낮을 수록 좋은 지표를 뜻합니다.


코어 웹 바이탈의 경우 다 사용자 경험과 연관성이 있다. 사용자 경험을 향상시키는 것이 왜 중요할까?

페이지 로드 시간이 길수록 이탈률에 큰 영향을 미칩니다.

  • 페이지 로드 시간이 1초에서 3초로 늘어나면 이탈률은 32% 증가합니다.
  • 페이지 로드 시간이 1초에서 6초로 늘어나면 이탈률은 106% 증가합니다.

사용자 경험 이외에 더 좋은 점이 있을까?

“SEO에 중요하다.”

구글이 자체적으로 진행했던 연구 결과에 따르면 Core Web Vital의 기준을 충족한 웹 페이지의 경우 방문자가 사이트에서 이탈할 가능성이 24%나 낮다고 합니다.
